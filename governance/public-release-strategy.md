# Frasberg AI Public Release Strategy

## Phase 1 — Internal Stabilization

- Finalize governance/legal pack
- Lock protected branches
- Complete codex architecture
- Validate multi-region deployment
- Prepare public documentation

## Phase 2 — Controlled Exposure

Make public:

- `FrasbergAI/docs`
- `FrasbergAI/api`
- `FrasbergAI/portal` (public version)
- SDKs and client libraries

Keep private:

- `FrasbergAI/luchii`
- `FrasbergAI/governance`
- `FrasbergAI/legal`
- Internal domain repos

## Phase 3 — Public Launch

Actions:

- Publish API Reference
- Release SDKs
- Announce Luchii
- Open developer portal
- Provide onboarding materials
- Enable public API key creation

## Phase 4 — Community Growth

- Open-source selected tools
- Publish governance summaries
- Accept community contributions
- Expand documentation
- Host developer events
