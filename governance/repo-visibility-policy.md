# Frasberg AI Repo Visibility Policy

## Purpose

Define when Frasberg AI repositories should be private, public, or selectively exposed.

## 1. Private Repositories (Default)

Use private visibility for:

- Core model repos (e.g., `luchii`)
- Governance, legal, and safety documents
- Multi-region deployment blueprints
- Internal tooling and MCP servers
- Brand protection and trademark assets
- Pre-release API specifications
- Experimental or unstable features

Reason: Protect IP, architecture, governance, and safety systems during development.

## 2. Public Repositories (Launch Phase)

Use public visibility for:

- Finalized API documentation
- Public SDKs
- Portal UI (public version)
- Developer onboarding materials
- Public governance summaries (not full constitutions)
- Open-source tools intended for community use

Reason: Enable developer adoption, transparency, and ecosystem growth.

## 3. Hybrid Repositories

Use private -> public transition for:

- Repos that begin as internal but become stable
- Repos that require community contributions
- Repos that support public APIs

Reason: Controlled exposure after stabilization.

## 4. Visibility Change Requirements

Before making any repo public:

- Governance Council approval
- Brand Protection review
- Safety Charter compliance
- Removal of internal-only documents
- Security scan
- License validation
- Release readiness checklist passed
