## 1. DRIFT CLASSIFICATION GRID

### Type-1 Drift — Documentation Drift
**Occurs when:**
- Governance docs diverge from constitutional state
- Certificates or memos mismatch anchor SHA
- Directory manifests lose parity

**Severity:** Low  
**Response:** Immediate doc parity restoration

### Type-2 Drift — Branch Drift
**Occurs when:**
- Default branch diverges from governed anchor
- Region branches lose parity
- Unauthorized commits appear

**Severity:** Medium  
**Response:** Branch reconciliation + audit entry

### Type-3 Drift — Routing Drift
**Occurs when:**
- Multi-region routing becomes inconsistent
- Autonomy routing deviates from anchor SHA
- Governance routing loses constitutional alignment

**Severity:** High  
**Response:** Routing map recalibration + stability envelope refresh

### Type-4 Drift — Stability Drift
**Occurs when:**
- SIM-series checks fail
- Parity checks fail
- Lockdown integrity weakens

**Severity:** Critical  
**Response:** Full stability envelope reinitialization

### Type-5 Drift — Autonomy Drift
**Occurs when:**
- Anchor SHA coherence breaks
- Execution records diverge
- Audit chain continuity is compromised

**Severity:** Maximum  
**Response:** Constitutional intervention + autonomy field reset
