# FrasbergAI / Luchii — Core Codex

Luchii is the nucleus of the FrasbergAI ecosystem: a governed, multi-domain architecture codex that defines shared structure, semantic nodes, and core contract scaffolds for future domain repositories.

This repository is the **central codex**, not a runtime engine. It exists to document governed lineage, codify multi-domain mesh boundaries, and provide the scaffold pack that later repositories can inherit.

## Repository Structure

```text
/docs             Technical documentation and API contracts
/docs/adr         Architecture decision records
/blueprints       Architectural and cosmological scaffolds
/diagrams         Visual system maps and schematics
/governance       Constitution, policies, and repository safeguards
/legal            Licensing and enterprise usage materials
/luchii           Initial TypeScript model-routing scaffold
/infra            Integration modules such as GitHub App auth
/schemas          Versioned configuration schemas
/test             Focused TypeScript regression tests
/portal           Portal and interface design scaffolds
/scripts          Automation and deployment utilities
/chapters         Narrative and conceptual documentation
/species          Entity and ontology definitions
/civilizations    Cultural and systemic models
```

## Codex Intent

- **Central codex:** Luchii defines the shared framing for the FrasbergAI mesh.
- **Multi-domain mesh:** each domain operates as a semantic node governed by a common lineage and core contract.
- **Grounded roadmap:** runtime and service repositories may emerge later, but this repository does not claim runtime capabilities that do not yet exist.

## Emerging Domain Pattern

The current scaffold anticipates future domain repositories such as:

- `/species` → `frasbergai-species`
- `/civilizations` → `frasbergai-civilizations`
- `/chapters` → `frasbergai-chronicles`

Runtime and service repositories can be introduced separately once the codex and domain boundaries stabilize.

## Roadmap

- **v1.0** — Core scaffolds committed
- **v1.1** — Domain split and repo orchestration
- **v2.0** — FrasbergAI Portal and unified SDK
- **v3.0** — Multi-region governance and production deployment
